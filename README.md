# SmartBike: Demand-Aware Rental & Management System

**Course:** COMP 202 - Software Engineering | Spring 2026
**Institution:** Abdullah Gül University (AGÜ)

## Project Overview
SmartBike is a web-based rental platform designed to digitize bike sharing while optimizing city operations through data analytics. Unlike standard applications, it integrates environmental and demand data to provide users with weather-based rental suggestions and administrators with demand-aware insights.

## First Iteration Notes (Current Progress)
The primary goal of this iteration was to establish the system architecture and core infrastructure. The following requirements have been defined for this stage:

* **Req1 - User Authentication:** Implementation of a secure Sign-up and Login system.
* **Req2 - Relational Database Design:** A schema consisting of at least 5 distinct tables (Users, Bikes, Zones, Reservations, and DemandData) with proper foreign key mapping.
* **Req3 - Zone Capacity Management:** Ability to list different city zones (Midtown, OldTown, etc.) and monitor their bike capacities.
* **Req4 - Environmental Data Integration:** Importing the `city_bike_demand_2025.csv` dataset to enable future smart-analytics features.
* **Req5 - Core UI Layout:** Development of the primary navigation structure, including Profile, Bike Reservation, and Smart Insights tabs.
